> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 외부 기술 아티클 URL 큐레이션 피드 — CRUD + 추천/북마크/댓글 + OG 파싱 + 알림/XP 연동

### Task 4: OgMetaParser (SSRF 방지)

**생성:** `feed/OgMetaParser.java`

@Component. 메서드: `OgMeta parse(String url)`

OgMeta record: `title, description, ogImageUrl, domain`

구현:
1. URL 스킴 체크 (http/https만)
2. DNS 해석 → IP가 private 대역이면 거부
3. Jsoup.connect(url).timeout(3000).maxBodySize(1_000_000).followRedirects(false).get()
4. meta[property=og:title/description/image] 파싱
5. domain 추출: new URL(url).getHost()
6. 파싱 실패 시 OgMeta(null, null, null, domain)

private IP 체크: `InetAddress.getByName(host)` → `isSiteLocalAddress() || isLoopbackAddress() || isLinkLocalAddress()`

### Task 5: FeedService

**생성:** `feed/FeedService.java`

의존성: FeedRepository, FeedCommentRepository, FeedLikeRepository, FeedBookmarkRepository, UserRepository, OgMetaParser, XpService, ApplicationEventPublisher

메서드:
- `create(userId, request)` — OG 파싱 + 저장 + XP 부여
- `getFeeds(category, sort, pageable, currentUserId)` — 목록 + myLiked/myBookmarked 배치 조회
- `getFeed(feedId, currentUserId)` — 상세
- `delete(feedId, userId)` — 본인만 soft-delete
- `like(feedId, userId)` — find-and-restore 패턴 + likeCount 원자적 업데이트 + 알림 + XP
- `unlike(feedId, userId)` — soft-delete + likeCount 감소
- `bookmark(feedId, userId)` — 저장 + bookmarkCount 증가
- `removeBookmark(feedId, userId)` — hard-delete + bookmarkCount 감소
- `getBookmarks(userId, pageable)` — 내 북마크
- `createComment(feedId, userId, request)` — 저장 + commentCount 증가 + 알림
- `getComments(feedId, pageable)` — 댓글 목록
- `deleteComment(feedId, commentId, userId)` — 본인만 soft-delete + commentCount 감소

like의 likeCount 원자적 업데이트: `@Modifying @Query("UPDATE TechFeed f SET f.likeCount = f.likeCount + :delta WHERE f.id = :id")`
