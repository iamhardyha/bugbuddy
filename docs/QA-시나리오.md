## 1. 인증 (Authentication)

### TC-AUTH-001: GitHub OAuth 로그인
**사전조건:** 비로그인 상태  
**절차:**
1. 헤더의 "GitHub로 로그인" 버튼 클릭
2. GitHub 인증 페이지에서 승인
3. `/oauth/callback`으로 리다이렉트

**기대결과:**
- [ ] 로그인 버튼이 사라지고 아바타 + 닉네임 + 로그아웃 버튼 표시
- [ ] 채팅 아이콘 (안읽음 뱃지 포함) 표시
- [ ] localStorage에 `accessToken`, `refreshToken` 저장됨
- [ ] 페이지 새로고침 후에도 로그인 상태 유지

### TC-AUTH-002: 로그아웃
**사전조건:** 로그인 상태  
**절차:** 헤더의 "로그아웃" 버튼 클릭

**기대결과:**
- [ ] 아바타/닉네임이 사라지고 "GitHub로 로그인" 버튼 표시
- [ ] localStorage에서 토큰 삭제됨
- [ ] 보호된 페이지(/chat, /notifications 등) 접근 시 홈으로 리다이렉트

### TC-AUTH-003: 토큰 만료 자동 갱신
**사전조건:** accessToken 만료, refreshToken 유효  
**절차:** 인증 필요한 API 호출 (예: 질문 작성)

**기대결과:**
- [ ] 401 응답 후 자동으로 `/api/auth/refresh` 호출
- [ ] 새 토큰 발급 후 원래 요청 재시도
- [ ] 사용자는 끊김 없이 작업 계속

### TC-AUTH-004: 토큰 완전 만료 (refresh도 만료)
**사전조건:** 두 토큰 모두 만료  
**절차:** 인증 필요한 API 호출

**기대결과:**
- [ ] localStorage에서 토큰 자동 삭제
- [ ] 로그인 버튼으로 전환
- [ ] `/`로 리다이렉트

### TC-AUTH-005: 비로그인 상태에서 보호 페이지 직접 접근
**절차:** URL 직접 입력으로 `/chat`, `/notifications`, `/settings/profile`, `/questions/new`, `/mentor/apply` 접근

**기대결과:**
- [ ] 모두 `/`로 리다이렉트
- [ ] 에러 없이 정상 처리

## 2. 질문 (Questions)

### TC-Q-001: 질문 목록 조회 (비로그인)
**절차:** 메인 페이지 접속

**기대결과:**
- [ ] 질문 카드 목록 표시 (제목, 카테고리, 태그, 조회수, 작성자, 시간)
- [ ] "질문하기" 버튼 미표시 (비로그인)
- [ ] 카테고리 필터 동작 (백엔드, 프론트엔드, DevOps 등)
- [ ] 상태 탭 동작 (전체, 진행중, 해결됨, 마감됨)
- [ ] "더 보기" 페이지네이션 동작
- [ ] 오른쪽 사이드바에 환영 메시지 표시 (비로그인)

### TC-Q-002: 질문 목록 조회 (로그인)
**사전조건:** 로그인 상태

**기대결과:**
- [ ] "질문하기" 버튼 표시 (인라인 + FAB)
- [ ] FAB는 인라인 버튼이 스크롤 밖으로 나가면 표시
- [ ] 오른쪽 사이드바에 프로필 카드 (닉네임, XP 프로그레스) 표시
- [ ] 멘토가 아닌 경우 "멘토 신청하기" CTA 카드 표시
- [ ] 이미 멘토인 경우 CTA 카드 미표시

### TC-Q-003: 질문 작성
**사전조건:** 로그인 상태  
**절차:**
1. "질문하기" 클릭 → `/questions/new`
2. 제목 입력 (최대 120자, 글자 수 표시 확인)
3. 카테고리 선택 (필수)
4. 질문 유형 선택 (필수)
5. 마크다운 본문 작성
6. 이미지 붙여넣기/드래그 업로드 테스트
7. 태그 입력 (최대 5개, Enter/콤마로 추가, X/Backspace로 삭제)
8. "질문 등록" 클릭

**기대결과:**
- [ ] 필수 필드 미입력 시 인라인 Alert 표시
- [ ] 태그 5개 초과 시 추가 불가
- [ ] 이미지 업로드 후 마크다운에 `![](url)` 삽입
- [ ] 성공 시 `/questions/:id`로 리다이렉트
- [ ] 작성자에게 +5 XP 부여

### TC-Q-004: 질문 상세 조회
**절차:** 질문 카드 클릭

**기대결과:**
- [ ] 제목, 본문(마크다운 렌더링), 카테고리, 태그, 조회수, 작성일 표시
- [ ] 작성자 프로필 사이드바 표시
- [ ] 유사 질문 추천 (최대 3개) 표시
- [ ] 조회수 1 증가 (새로고침 시 반영)
- [ ] 브레드크럼 네비게이션 동작

### TC-Q-005: 질문 수정 (작성자)
**사전조건:** 질문 작성자로 로그인  
**절차:**
1. 질문 상세에서 "수정" 버튼 클릭 → `/questions/:id/edit`
2. 제목/본문/카테고리/태그 수정
3. "수정 완료" 클릭

**기대결과:**
- [ ] 기존 데이터가 폼에 사전 입력됨
- [ ] 수정 성공 후 질문 상세로 리다이렉트
- [ ] 수정된 내용 반영 확인

### TC-Q-006: 질문 수정 — 비작성자/마감 상태 차단
**기대결과:**
- [ ] 다른 유저가 수정 시도 → 403 에러
- [ ] 마감된 질문의 수정 페이지 접근 → 질문 상세로 리다이렉트

### TC-Q-007: 질문 삭제 (작성자)
**절차:** 질문 상세에서 "삭제" 버튼 클릭

**기대결과:**
- [ ] 확인 다이얼로그 (danger 스타일) 표시
- [ ] "취소" 클릭 → 아무 일 없음
- [ ] "삭제" 확인 → 질문 소프트 삭제, 목록에서 사라짐

### TC-Q-008: 질문 마감 (작성자)
**절차:** 질문 상세에서 "마감" 버튼 클릭

**기대결과:**
- [ ] 확인 다이얼로그 (warning 스타일) 표시
- [ ] 확인 후 상태 `CLOSED`로 변경
- [ ] "수정", "마감" 버튼 사라짐
- [ ] 답변 작성 폼 사라짐
- [ ] 이미 마감된 질문에 중복 마감 → 400 에러

## 3. 답변 (Answers)

### TC-A-001: 답변 작성
**사전조건:** 로그인 상태, OPEN 상태 질문  
**절차:**
1. 질문 상세 하단 답변 작성 폼에 본문 입력 (최소 10자)
2. 멘토인 경우 "1:1 멘토링 가능" 토글 확인
3. 이미지 업로드 테스트
4. "답변 등록" 클릭

**기대결과:**
- [ ] 답변 목록에 새 답변 추가
- [ ] 작성자에게 +5 XP
- [ ] 질문 작성자에게 "새 답변" 알림 발송
- [ ] 멘토가 아닌 유저의 `allowOneToOne`은 항상 false 저장

### TC-A-002: 답변 채택 (질문 작성자만)
**사전조건:** 질문 작성자로 로그인, OPEN 질문  
**절차:** 답변의 "채택" 버튼 클릭

**기대결과:**
- [ ] 확인 다이얼로그 표시
- [ ] 답변에 "채택된 답변" 뱃지 표시
- [ ] 질문 상태 `SOLVED`로 변경
- [ ] 답변 작성자에게 +30 XP
- [ ] 답변 작성자에게 "답변 채택" 알림 발송
- [ ] 이미 채택된 질문에 중복 채택 → 400 에러
- [ ] 다른 유저가 채택 시도 → 403 에러

### TC-A-003: 도움됐어요 리액션
**사전조건:** 로그인 상태  
**절차:** 다른 유저의 답변에 "도움됐어요" 클릭

**기대결과:**
- [ ] 카운트 +1, 버튼 활성 상태로 변경
- [ ] 답변 작성자에게 +20 XP
- [ ] 답변 작성자에게 "도움됐어요" 알림 발송
- [ ] 다시 클릭하면 리액션 취소 (카운트 -1)
- [ ] 자신의 답변에 리액션 → 400 에러
- [ ] 중복 리액션 → 409 에러

### TC-A-004: 답변 수정/삭제 (작성자)
**기대결과:**
- [ ] 수정: 본문 변경 후 저장 → 반영 확인
- [ ] 삭제: 확인 다이얼로그 후 소프트 삭제
- [ ] 비작성자에게 수정/삭제 버튼 미표시

## 4. TechFeed

### TC-F-001: 피드 목록 조회
**절차:** TechFeed 탭 클릭 (`/feeds`)

**기대결과:**
- [ ] 피드 카드 목록 표시 (OG 이미지, 제목, 카테고리, 작성자, 좋아요/댓글 수)
- [ ] OG 이미지 없는 경우 카테고리 이름 그래디언트 폴백
- [ ] 카테고리 필터 동작
- [ ] 정렬 동작 (최신순/인기순)

### TC-F-002: 피드 작성
**사전조건:** 로그인 상태  
**절차:**
1. FAB "피드 등록" 클릭 → `/feeds/new`
2. URL 입력 (필수)
3. 카테고리 선택 (필수)
4. 추천 이유 입력 (최대 500자)
5. "피드 등록" 클릭

**기대결과:**
- [ ] 서버에서 URL의 OG 메타데이터 자동 파싱
- [ ] 성공 시 `/feeds/:id`로 리다이렉트
- [ ] OG 파싱 실패 시 `FEED_OG_PARSE_FAILED` 에러 표시

### TC-F-003: 피드 상세 — 좋아요/북마크
**절차:**
1. 피드 상세 페이지 접속
2. "추천" 버튼 클릭
3. "북마크" 버튼 클릭

**기대결과:**
- [ ] OG 프리뷰 카드 + "원문 보기" 버튼 동작
- [ ] 좋아요 토글: 카운트 즉시 업데이트 (Optimistic UI)
- [ ] 북마크 토글: 즉시 반영
- [ ] API 실패 시 롤백
- [ ] 자기 피드에 좋아요 → 400 에러
- [ ] 비로그인 시 좋아요/북마크 UI 미표시

### TC-F-004: 피드 댓글
**절차:**
1. 댓글 입력 + "댓글 등록" 클릭
2. 본인 댓글에 삭제 버튼 클릭

**기대결과:**
- [ ] 댓글 목록에 즉시 추가
- [ ] 피드 작성자에게 "새 댓글" 알림 발송
- [ ] 본인 댓글에만 삭제 버튼 표시
- [ ] 삭제 후 목록에서 즉시 제거
- [ ] 20개 이상 시 페이지네이션

### TC-F-005: 북마크 목록
**절차:** `/feeds/bookmarks` 접속

**기대결과:**
- [ ] 북마크한 피드만 표시
- [ ] 클릭 시 피드 상세로 이동
- [ ] 비로그인 시 `/`로 리다이렉트

## 5. 1:1 채팅 (Chat)

### TC-CHAT-001: 채팅 제안 (멘티 → 멘토)
**사전조건:** 질문 작성자 로그인, 멘토의 답변에 `allowOneToOne=true`  
**절차:** 답변 카드의 "1:1 채팅 신청" 버튼 클릭

**기대결과:**
- [ ] 채팅방 생성 (PENDING 상태)
- [ ] 멘토에게 "채팅 요청" 알림 발송
- [ ] 같은 멘토/멘티/질문 조합 중복 신청 → 409 에러
- [ ] `allowOneToOne=false` 답변에는 버튼 미표시
- [ ] 질문 작성자가 아닌 유저에게 버튼 미표시

### TC-CHAT-002: 채팅 수락 (멘토)
**사전조건:** 멘토 로그인, PENDING 채팅방 존재  
**절차:** 채팅방에서 "수락" 버튼 클릭

**기대결과:**
- [ ] 확인 다이얼로그 표시
- [ ] 채팅방 상태 PENDING → OPEN 전환
- [ ] 시스템 메시지 "멘토링 세션이 시작되었습니다." 표시
- [ ] 멘티에게 "채팅 수락" 알림 발송
- [ ] 멘티 채팅 목록에서 실시간 상태 업데이트 (STOMP)

### TC-CHAT-003: 실시간 메시지 송수신
**사전조건:** OPEN 상태 채팅방, 양쪽 유저 접속  
**절차:**
1. 유저 A가 메시지 입력 + Enter
2. 유저 B 화면에서 실시간 수신 확인
3. 유저 B가 답장

**기대결과:**
- [ ] 메시지 즉시 상대방에게 전달 (WebSocket)
- [ ] 메시지 버블에 발신자 닉네임 + 시간 표시
- [ ] 연결 상태 아이콘 초록색 (WiFi)
- [ ] Shift+Enter로 줄바꿈 가능
- [ ] 전송 후 입력창 비워짐
- [ ] 자동 스크롤

### TC-CHAT-004: 읽음 처리 + 안읽음 카운트
**절차:**
1. 유저 A가 메시지 전송
2. 유저 B의 채팅 목록에서 안읽음 카운트 확인
3. 유저 B가 해당 채팅방 입장

**기대결과:**
- [ ] 채팅 목록에 안읽음 숫자 뱃지 표시
- [ ] 채팅방 입장 시 `markAsRead` 호출
- [ ] 안읽음 카운트 0으로 리셋

### TC-CHAT-005: 채팅 종료 + 피드백
**절차:**
1. 채팅방에서 "종료" 버튼 클릭
2. 확인 다이얼로그에서 "종료" 확인
3. 피드백 모달에서 별점(1-5) + 코멘트 입력
4. "제출" 클릭

**기대결과:**
- [ ] 채팅방 상태 OPEN → CLOSED 전환
- [ ] 시스템 메시지 "멘토링 세션이 종료되었습니다." 표시
- [ ] 피드백 모달 자동 오픈 (종료한 당사자만)
- [ ] 별점 필수, 코멘트 선택
- [ ] 제출 후 "피드백 작성" 버튼 사라짐
- [ ] 상대방은 CLOSED 상태를 보고 직접 "피드백 작성" 버튼 클릭
- [ ] 중복 피드백 제출 → 409 에러

## 13. 보안/엣지케이스

### TC-SEC-001: 인증 우회 시도
- [ ] Authorization 헤더 없이 보호 API 호출 → 401
- [ ] 만료된 토큰으로 호출 → 401
- [ ] 변조된 토큰으로 호출 → 401
- [ ] 유저 토큰으로 `/api/admin/*` 접근 → 403

### TC-SEC-002: 권한 우회 시도
- [ ] 다른 유저의 질문 수정/삭제 → 403
- [ ] 다른 유저의 답변 수정/삭제 → 403
- [ ] 비참여자가 채팅방 메시지 조회 → 403
- [ ] 비참여자가 채팅방 종료 시도 → 403
- [ ] 다른 유저의 알림 읽음 처리 → 403

### TC-SEC-003: 입력 유효성
- [ ] 빈 문자열/null 입력 → 400 에러
- [ ] 필드 길이 초과 → 400 에러
- [ ] 잘못된 enum 값 → 400 에러
- [ ] 존재하지 않는 ID → 404 에러
- [ ] XSS 시도 (`<script>alert(1)</script>`) → 마크다운 렌더러에서 이스케이프

### TC-SEC-004: 중복 액션 방지
- [ ] 같은 질문에 같은 멘토/멘티 채팅 중복 → 409
- [ ] 같은 답변에 같은 유저 리액션 중복 → 409
- [ ] 같은 피드에 같은 유저 좋아요 중복 → 409
- [ ] 같은 피드에 같은 유저 북마크 중복 → 409
- [ ] 같은 대상에 같은 유저 신고 중복 → 409
- [ ] 같은 채팅방에 같은 유저 피드백 중복 → 409

### TC-SEC-005: WebSocket 보안
- [ ] 토큰 없이 STOMP CONNECT → 연결 거부
- [ ] 유효하지 않은 토큰으로 CONNECT → 연결 거부
- [ ] OPEN이 아닌 방에 메시지 전송 → 에러
